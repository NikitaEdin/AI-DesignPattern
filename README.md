# LLM Efficiency in Code Design Pattern Identification
Honours research conducted by Nikita Lanetsky at Edinburgh Napier University.<br>
AI-driven static analysis for design pattern recognition, emphasised on the latest state of LLM and its efficiency.

> [!NOTE]  
Project setup and usage details will be shared upon completion, as the project is undergoing frequent updates and modifications.

## Abstract
This Honours project researches the ability and efficiency of the current state of Large Language Models (LLMs) in identifying implemented design patterns within source code.<br>
The research aims to evaluate four prominent top-leading LLM platforms - Ollama, OpenAI, Claude, and the recent rise of Kimi K2 - Through systemic testing suing algorithmically generated and evaulated code samples containing various level of complexity in code and design patterns.

## Research Objectives
- **Primary Objective**: Quantify and compare the accuracy and efficiency of different LLMs in recognising common software design pattern in code implementations.
- **Secondary Objectives**: 
- - Analyse pattern recognition performance across difference code complexities.
- - Evaluate the impact of single and multi-layered algorithems on identification accuracy.
- - Develop benchmarking methodologies for LLM code analysis capabilities.

### Design Patterns Under Inverstigation
13 Patterns in total.

- Creational Patterns
- - Singleton, Factory, Builder, Prototype
- Structural Patterns
- - Adapter, Decorator, Facade, Proxy
- Behavioural Patterns
- - Observer, Strategy, Command, Iterator, State
