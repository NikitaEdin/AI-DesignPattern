## Naming Convention

Files follow: **`<Pattern>_<ID>_<Difficulty>_<LLM-Prefix>.py`**

- **`<Pattern>`**: Design pattern name (e.g., `Adapter`, `Singleton`).
- **`<ID>`**: Zero-padded sequential index (e.g., `0`, `1`, `2`, ... ) for multiple generations per pattern/difficulty/LLM.
- **`<Difficulty>`**: Complexity level:
  | Level | Description |
  |-------|-------------|
  | `E`   | Easy: Basic core elements. |
  | `M`   | Medium: Standard implementation + features. |
  | `H`   | Hard: Complex with optimisations, edge cases. |
- **`<LLM-Prefix>`**: Short identifier for the generating LLM (from `shared/llm_interface.LLMFactory` prefixes):

    | Prefix   | LLM Provider     |
    |----------|------------------|
    | `O`      | OpenAI          |
    | `C`      | Claude          |
    | `GROK`   | Grok            |
    | `Q3C`    | Qwen            |
    | `GROK4F` | Grok4Fast       |
    | `KimiK2` | KimiK2          |

**Examples**:
- `Adapter_0_E_C.py`: Adapter pattern, 1st generation (ID=0), Easy, generated by Claude.
- `Singleton_42_M_O.py`: Singleton, 43rd gen, Medium, OpenAI.
- `Factory_5_H_GROK.py`: Factory, ID=5, Hard, Grok.
